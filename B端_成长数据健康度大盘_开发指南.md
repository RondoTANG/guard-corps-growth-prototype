# B端成长数据健康度大盘 - 开发指南

本文档用于向前端与后端开发团队说明“成长数据健康度大盘”页面的显示、筛选、统计口径与交互逻辑，确保最终研发交付与产品设计保持高度一致。

---

## 1. 页面整体布局与联动机制
大盘整体采用**“左树右数”**的两栏布局模式：
- **左侧边栏 (组织架构树)**：是全局唯一的组织维度筛选器。支持多选、层级穿透、搜索。**（注：数据的统计来源严格关联用户账号上所绑定的“所在党委”、“下属党委”、“党支部”三级真实组织结构）**。
- **右侧主内容区**：数据可视化大盘，从上到下依次为：核心指标卡片、概览占比图表、横向对比图、流转漏斗图、底层流水明细表。

**联动原则：**
右侧所有的图表、指标和表格，其数据请求的参数都**必须携带左侧选中的组织节点 ID 集合**以及顶部的**全局时间范围**，实现数据的100%强联动。

---

## 2. 组织树交互与降维逻辑

### 2.1 父子级联关系
- 选中父节点时，默认应在底层逻辑中包含其所有子节点的用户数据进行聚合计算。

### 2.2 自动层级下钻 (Smart Drill-down)
- 图表【活跃与段位分布对比】的“自动”下拉选项，依赖左侧树的选中状态。
- **特例处理**：如果用户仅选中了最顶层单一节点（如“集团党委”），1 个节点无法形成横向对比。此时，前端/后端逻辑需**自动向下钻取一层**，提取其下一级的所有子组织（如各大二级党委）作为 X 轴数据进行渲染。

---

## 3. 图表统计与业务逻辑

### 3.1 顶部核心指标 (KPI卡片)
- **成功晋级人次**：在所选时间和组织范围内，成功晋升更高段位的人次总和。卡片下方需动态显示其中晋升至高段位（L3/L4）的具体数值。
- **保级失败降级人次**：保级期满未达标导致段位下滑的人次总和。
- **同比/环比**：需展示较上周期的变化百分比（红色代表恶化或下降，绿色代表增长或向好，需结合业务语境。例如降级人次增加为负面，应标红）。

### 3.2 组织晋升与降级分布分析 (分类柱状图)
这是展示大盘段位健康度的核心图表。
- **X轴分类**：L1➞L2, L2➞L3, L3➞L4 (晋升通道)；L4➞L3, L3➞L2, L2➞L1 (降级通道)；外加 保级缓冲、荣誉保底。
- **计费口径 (核心)**：
  - 统计维度必须严格为**“人次”**。
  - **跨级处理规则**：若同一用户发生跨级跃升（如从 L1 直接跃升至 L3），不能只记1次，必须在【L1 ➞ L2】与【L2 ➞ L3】各计 1 人次。降级跨级同理。这样才能保证业务逻辑的流转账本完全算平。

### 3.3 活跃与段位分布对比 (双Y轴图)
- 柱状图部分（堆叠）：展示每个组织内部，L1~L4 四个段位的人数占比（堆叠至100%）。
- 折线图部分：展示该组织的整体活跃度百分比。
- **数据聚合与展示维度规则（核心算法：木桶效应向下对齐）**：
  - **行政级别判定**：判定组织的“层级”时，**绝对不能依赖前端树形控件的 DOM 嵌套深度**。必须严格根据节点的徽章标识（如“二级”、“三级”）或节点类型（如“二级党委”、“党支部”）来推断真实的业务行政级别。
  - **统一高维**：系统优先尝试将所有选中分支向真实的“最高全选行政层级”对齐（例如优先在三级齐平）。
  - **向下兼容**：若某分支拥有该层级组织，但只处于“半选”状态（即只勾选了部分下级），则由该组织代为出战，但图表数据仅统计其内部真实被勾选的下级数据。
  - **木桶下钻兜底**：若检测到某些特殊架构分支（例如二级直接越级挂载五级支部，导致该分支**物理缺失**三/四级）**根本不存在**该基准层级，则“统一高维对齐”失效。此时触发下钻机制，全体寻找大家共同拥有的最深层级（木桶短板）进行重新对齐。确保不管物理架构如何畸形，最终呈现在X轴上的柱子一定是“跨分支完全平级（同级别 PK）”的横向对比。
  - **图表X轴与Tooltip名称显示规则**：针对由于跨级混编结构导致的“强制同台对比”（例如：某个缺少党支部的“三级党委”与另一个独立的“党支部”在X轴作为并列柱子），为了防止业务理解歧义，**前端要求在 X 轴标签和 Hover 提示框的名称后，自动附加上该节点真实的行政层级后缀标识**。例如：展示为 `A党委 (三级)` 和 `B党支部 (党支部)`。

## 4. 底层 XP 资产流水明细表
该表格作为明细对账使用。
- **业务类型枚举**：
  必须包含完整的XP流转类型，每种类型对应不同的颜色标识：
  - `完成作业/任务` (正向收入，绿色标识)
  - `自动过期失效` (自然扣除，灰色标识)
  - `违规扣减` (惩罚性扣除，红色标识)
  - `人工干预/调账` (特殊修正，黄色标识)
- **前端交互规范**：
  由于表格字段较多，表格内部可支持 X 轴横向滚动；但**底部的分页器（Pagination）必须放在滚动容器的外部**，确保用户在不左右拖动表格的情况下，能随时点击下一页。

## 5. 核心交互规范与字段统计说明

### 5.1 左侧树容器自适应与拖拽缩放 (Resizable Sidebar)
- **拖拽伸缩**：由于党组织名称可能出现极长的情况，左侧组织架构树容器右边界需提供拖拽条（Resizer）。用户鼠标按住后可左右拖拽以调节容器宽度。
- **边界限制**：建议最小宽度限制为 `240px`，最大宽度限制为屏幕宽度的 `40%` 或 `500px`，超过部分文字允许 `ellipsis` 截断或横向滚动条。

### 5.2 核心漏斗转化指标口径
- **任务参与率** = `该段位在周期内提交过作业的用户数 / 该段位总用户数`
- **作业通过率** = `审核通过的作业数 / 该段位提交的总作业数`
*(注：前端需在相应图表的图例或提示词（Tooltip）中透出该统计公式，以降低业务理解成本。)*

### 5.3 活跃指标定义
- 大盘中提到的“成长活跃用户”，统一口径为：**在当前所选时间范围内，有任何 XP 获取或消耗行为的用户**（即只要有非“0”的账单流水产生即记为活跃，不含系统自动扣除）。

## 6. 生态巡航分析智能体 (Eco-Cruiser AI) 交互与调优说明

为了辅助业务运营，右侧大盘上方设有一个【唤醒 AI 诊断】的入口。

### 6.1 唤醒交互
- 用户点击后，从页面右侧滑出 AI 智能体抽屉（Drawer），并带半透明黑色蒙版（Overlay）。
- 在面板打开时，前端应通过骨架屏或 Spinner 动画模拟 AI 正在“实时读取 T-1 数据”。

### 6.2 动态 Prompt 与推演逻辑
- 前端需配合后端传递当前时间节点（月初/月中/月末）。
- 智能体会根据时间节点加载不同维度的洞察（比如：月初维稳，月末防流失）。
- 面板内包含【期望占比推演】设置区，运营可动态拖动滑块调整期望的段位分布，点击“重新推演”后，更新大盘的策略建议。

*(注：该智能体的完整设计逻辑与参数构建，详见 `03_生态巡航智能体构建方案.md`)*
