# 东风护卫军原型项目全局规则 (Project-Scoped Rules)

## 1. UI 与组件级一致性 (可随项目迭代持续补充)
### 1.1 导航与侧边栏
* **结构严格统一**：所有 B 端后台页面（`B端_*.html`）的左侧导航菜单，其 HTML DOM 结构必须完全一致。
* **高亮状态唯一对应**：当前访问的页面文件名，必须与菜单中带有 `active-menu` 样式的高亮文字严格对应。严禁出现“A 页面高亮 B 模块”。

### 1.2 待补充的校验规则 (Examples)
* *(未来添加)* **响应式适配**：所有弹窗必须包含响应式 `max-w-md` 及 `mx-4` 保护。
* *(未来添加)* **假数据规范**：禁止使用 Lorem Ipsum，必须使用贴合“东风日产护卫军”业务视角的真实假数据。

## 2. 自动化 QA 闭环（Agentic QA Loop）
在协助用户进行本目录下任何 B 端 HTML 原型的创建、大范围修改或样式同步任务即将结束时：
1. **强制自检**：你必须主动调用 `prototype-consistency-checker` 技能。
2. **静默修复**：自动执行包含 `--fix` 参数的脚本以抹平可能遗漏的高亮状态错误。
3. **交付验收**：只有当终端输出提示所有菜单一致性修复完成后，方可告知用户任务已完成。绝对不依赖用户的肉眼去排查这种低级错误。
